# Kali Prox Installer - Header Logo Prompt

## Create a unique, modern, and brandable combination mark logo for an open-source automation tool called "Kali Prox Installer".

The logo should combine a stylized Kali Linux dragon and Proxmox server rack/VM icon with the wordmark "Kali Prox Installer" in bold typography.

## Design Style

Technical yet approachable, suitable for a DevOps/Security automation tool.

## Color Palette

| Color | Hex | Usage |
|-------|-----|-------|
| Kali Blue | `#2B84D6` | Primary gradient start |
| Kali Black | `#1A1A2E` | Primary gradient end |
| Proxmox Orange | `#E57000` | Accent color |
| GitHub Dark | `#0D1117` | Background |
| White/Light Grey | `#F0F6FC` | Text |

- Gradient blend from blue to orange representing the integration of Kali and Proxmox
- White or light grey text on dark background

## Typography

- **"Kali Prox"**: Bold, geometric sans-serif for modern tech feel
- **"Installer"**: Monospace font (JetBrains Mono or Fira Code) to emphasize scripting/automation
- Clean and easily legible for GitHub repository header

## Icon Integration

- The Kali dragon silhouette may be integrated into the "K" or placed above the wordmark
- Proxmox-style server/VM element (simplified boxes or stacked layers) positioned beside or intertwined with the dragon
- Consider adding a subtle terminal/command line element (`>_`) to represent script automation

## Design Principles

- Flat and vector-based (no 3D), minimal yet powerful
- Clean lines, tech-forward aesthetic
- Suitable for both light and dark GitHub themes

## Background

- Dark (#0D1117 - GitHub dark mode) to make the gradient pop
- Subtle hex pattern or grid overlay for technical depth

## Overall Vibe

- Professional, security-focused, automated
- Conveying "one-command deployment" and "penetration testing lab builder"
- Ready for open-source community branding

## Additional Notes

- Ensure the dragon element is respectful of Kali Linux branding (don't over-modify)
- Keep the Proxmox reference subtle but recognizable to users
- Balance between security (Kali) and infrastructure (Proxmox)
